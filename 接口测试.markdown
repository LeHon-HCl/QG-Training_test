## 📌 使用说明

1. **替换动态参数**：测试前将表格中的 `:id`（如 `1`、`2`）替换为你数据库中实际存在的ID。
2. **Token 获取**：先执行登录接口，从响应中复制 `token`，后续请求在 Header 中添加 `Authorization: Bearer <token>`。
3. **权限验证**：重点验证班主任跨班操作、学生越权操作是否被正确拦截（返回 403）。
4. **数据准备**：
   - 确保 `users` 表有 `admin`、`teacher`、`student` 三种角色用户。
   - 确保班级、学生、教师已正确绑定（参考之前提供的 SQL）。

按此表格逐项测试，可全面验证系统功能完整性和权限控制正确性。
